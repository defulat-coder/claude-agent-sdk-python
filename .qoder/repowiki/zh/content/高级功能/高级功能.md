## 简介
本文件面向高级用户与工程团队，系统化梳理 Claude Agent SDK 的高级能力与最佳实践，覆盖以下主题：
- 系统提示与代理配置：动态修改、代理定义、设置源控制
- 输出格式控制：JSON 模式输出、思维令牌配置、输出样式定制
- 部分消息与流式更新：partial.json 支持、增量流式体验
- 预算控制与成本管理：最大预算阈值、错误类型识别
- 高级配置：性能调优、资源限制、权限与沙箱策略
- 多轮对话：上下文管理、状态保持、中断与任务控制
- 专家级模式：权限回调、MCP 服务器控制、会话检索与回放
- 性能监控与调试：速率限制事件、日志与解析器行为
- 与其他 AI 框架与工具的集成模式

## 项目结构

- 单次查询接口：适合一次性、无状态任务
- 流式客户端：适合多轮对话、实时交互、中断与动态控制

## 架构总览
SDK 在运行时通过子进程与 CLI 通信，采用“控制协议 + 流式消息”的方式实现双向交互。流式客户端在连接后启动读取任务组，持续消费消息并解析为强类型对象；单次查询在完成一次请求后关闭。

```mermaid

participant Client as "ClaudeSDKClient"
participant Query as "Query 控制协议"
participant Transport as "传输层(SubprocessCLITransport)"
participant Parser as "消息解析器"
participant CLI as "Claude Code CLI"
App->>Client : connect()
Client->>Transport : connect()
Client->>Query : start() + initialize()
Transport->>CLI : 启动子进程并写入初始化数据
CLI-->>Transport : 返回初始化结果
Transport-->>Client : 初始化完成
App->>Client : query(prompt)
Client->>Transport : 写入用户消息(JSONL 行)
Transport->>CLI : 发送消息
CLI-->>Transport : 流式返回消息
Transport-->>Parser : 解析消息
Parser-->>Client : 强类型消息
Client-->>App : 迭代返回消息
App->>Client : interrupt()/set_model()/set_permission_mode()
Client->>Query : 控制命令
Query->>CLI : 发送控制指令
CLI-->>Query : 状态反馈
```
